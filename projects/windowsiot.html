Roles

During my summer, my roles included:

  • Developing support for Touch Input over i2c from the PiTFT touchscreen (C++) into HID interface for Windows IoT Core running on the Raspberry Pi
  • Integrating Touch with an SPI Display Driver developed by teammembers for the PiTFT
  • Leading development for an example 3D Printer control application and authoring project documentation
  • Publishing project to Open Source community for IoT hobbiests, Windows enthusiasts, and makers
  • Won 1st Place at the Microsoft Explore Tech Fair for best overall project


Phases of Development

Requirements: knowing knowing the target customers and determining their wants and needs. The target audience for this project was the Windows IoT Community and the project was in response for requests for a touchscreen that could display and run Windows applications for the Pi

Planning: writing specifications, design plans, and architecture. In this phase, specifics of what modes would contribute to display – kernel mode Display Driver and user mode Touch Listener – and how they would talk to the display (over SPI for Driver and I2c for touch). Frameworks that would contribute to these modes were also part of the design plans. For the Display driver, already developed programs were incorporated – Windows Graphics and Windows 10 applications. For the touch listener, an already developed HID Stack was leveraged in building touch.

Implementation/Design: designing and implementing the hardware components and software. Spec sheets for the touchscreen guided the SPI and I2C connections and the display driver and touch input code was written specific to the Adafruit PiTFT.

Integration: various hardware and software pieces are brought together to make up the end-to-end functionality. The display developed by two interns in my team and touch (my work) were developed separately and as PM during the integration phase my task was to combine the display and touch to work together for the touchscreen display.

Testing: testing of end-to-end functionality. In this phase, my team tested our work for the touch display by getting Windows applications to run. We wrote a 3D printer Windows application that could show the display working and that a user could interact with via touch.

Delivery: product is delivered to customers for early adaptation and testing. As PM, I documented the project with instructions for users open sourcing it to the Windows IoT community for testing and use for their projects. Since August, the project has had over 3000 visitors from Windows developers and is being updated by engineers who have adopted the project for long-term updates
